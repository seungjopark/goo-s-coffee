# 구스커피 주문 관리 시스템 기획서

## 📋 프로젝트 개요

**프로젝트명**: 구스커피 주문 관리 시스템  
**목적**: 커피 원두 주문 관리의 디지털화 및 효율성 증대  
**작성일**: 2025년 10월 15일

## 🎯 현재 상황 및 문제점

### 현재 상황
- 문자와 전화를 통한 커피 원두 주문 접수
- 메모장을 이용한 수기 주문 기록

### 주요 문제점
- ❌ 주문 누락 발생
- ❌ 총 계산의 어려움
- ❌ 데이터 관리의 비효율성
- ❌ 월별/연별 매출 분석의 어려움

## 🎯 시스템 목표

1. **정확한 주문 관리**: 누락 없는 체계적인 주문 기록
2. **자동 계산**: 실시간 가격 계산 및 합계 산출
3. **효율적인 데이터 관리**: 일별/월별/연별 데이터 정리
4. **매출 분석**: 기간별 매출 현황 파악

## 🔧 핵심 기능 요구사항

### 1. 상품 관리
- **커피 원두 명칭 설정**: 원두 종류별 이름 입력/수정
- **가격 설정**: 원두별 단가 입력/수정
- **상품 목록 관리**: 원두 추가/삭제/수정

### 2. 주문 관리
- **수량 조절**: 플러스(+)/마이너스(-) 버튼으로 수량 조절
- **실시간 계산**: 단가 × 수량 = 개별 상품 총액
- **다중 상품 주문**: 여러 원두 동시 주문 가능
- **전체 합계**: 주문 내 모든 상품의 총 금액 계산

### 3. 데이터 관리
- **일별 기록**: 날짜별 주문 내역 저장
- **주문 내역 조회**: 과거 주문 검색 및 조회
- **수정/삭제**: 기록된 주문의 수정 및 삭제

### 4. 통계 및 결산
- **월별 결산**: 월 단위 매출 및 판매량 집계
- **연별 결산**: 연 단위 매출 및 판매량 집계
- **상품별 통계**: 원두별 판매 실적 분석

## 📊 데이터베이스 설계

### 1. 상품 테이블 (products)
```sql
- id (Primary Key)
- name (상품명)
- price (가격)
- created_at (생성일)
- updated_at (수정일)
```

### 2. 주문 테이블 (orders)
```sql
- id (Primary Key)
- order_date (주문일)
- total_amount (총 금액)
- created_at (생성일)
```

### 3. 주문 상세 테이블 (order_items)
```sql
- id (Primary Key)
- order_id (주문 ID, Foreign Key)
- product_id (상품 ID, Foreign Key)
- quantity (수량)
- unit_price (단가)
- subtotal (소계)
```
